# 软件设计师（中级）复习计划 - 2026年下半年

## 学习理念
- **结构化思维**：利用 Markdown 和 Mermaid 图表将知识点体系化。
- **以练促学**：通过 TypeScript 实现算法和设计模式，加深理解。
- **视觉化学习**：所有架构、流程和关系图均使用 Mermaid 绘制。
- **高效迭代**：关注重点和难点，不纠结于琐碎知识点。

## 复习阶段
1. **第一阶段：夯实基础（计算机组成、OS、网络、数据库）**
2. **第二阶段：核心突破（软件工程、结构化/面向对象分析与设计、设计模式）**
3. **第三阶段：专项攻克（算法与数据结构、法律法规、标准化）**
4. **第四阶段：冲刺实战（历年真题演练、查漏补缺）**

## 目录结构
- `notes/`: 核心知识点笔记
- `exercises/`: 算法与设计模式的代码实现
- `practice/`: 历年真题分析
- `syllabus/`: 大纲与进度跟踪

## C 档 · 每日 1～1.5h（约 7～10h/周）

> 适用：在职备考，优先过线；笔记求精不求全，时间优先给选择题与手算题。

### 时间分配（每周约 8～9h 均值）

| 用途 | 占比 | 约/h周 |
|------|------|--------|
| 当前主章（笔记 + 小题） | 55% | 4.5～5 |
| 选择题 / 手算 | 30% | 2.5～3 |
| 下午专项（8 月起） | 10% → 后期加重 | 0～2 |
| 回顾 / 错题 | 5% | 0.5 |

### C 档原则

- **笔记**：考点 + 一张图 + 1 道例题；不写百科。
- **练习**：PV、子网、范式、流水线等手算题 + 选择限时。
- **下午**：UML（类图/时序）+ 1 类算法题 + 设计模式认场景（6～8 个高频即可）。
- **真题**：近 2～3 年选择 + 下午按题型拆练；不求刷完 5 年。
- **法规 / 英语**：考前 2～3 周集中背表，平时不单独开章。

### 典型一天（1.5h）

- 45min 主章 + 30min 5～20 道选择 + 15min 错题；或 60min 手算 + 30min 回顾。
- 每周固定 **1 次** 整块 90min：下午半道题 **或** 选择 30 题限时。

### 六个月主题

```text
6月  计组真题收尾 + OS（进程/PV/内存）
7月  OS 查漏 + 网络（子网/TCP）+ 软工脑图
8月  数据库（范式/SQL/事务）+ UML 入门
9月  四门回顾 + 算法/排序查找 + 设计模式认名
10月 选择真题 + 下午拆题 + 法规英语表
11月  2 套限时 + 错题 + 速查表
```

### 6 月 checklist（约 30h）

- [ ] 计组选择 30～40 题限时，错题记入 [practice/README.md](./practice/README.md)
- [ ] OS：进程状态与调度（名词 + 对比）
- [ ] OS：PV — 生产者-消费者、读者-写者（`03-pv-semaphore.ts`）
- [ ] OS：内存 — 分页、LRU 手算 ≥2 题
- [ ] OS：文件/设备 — 磁盘调度 ≥1 题
- [ ] **月末**：PV 能独立写信号量；LRU 能对；计组自测 ≥24/40

### 7 月 checklist（约 30h）

- [ ] 网络：子网划分（隔日 1 题）
- [ ] 网络：TCP/UDP、三次握手 / 四次挥手（一张对比表）
- [ ] 网络：HTTP/DNS/常见端口（常考表）
- [ ] OS：死锁四条件 + 银行家 1 道例题
- [ ] OS + 网选择合计 ≥20 题/周，月末自测 ≥28/50
- [ ] 软工：生命周期、测试级别、维护类型（一张 Mermaid，每周 30min）

### 8 月 checklist（约 30h）

- [ ] 数据库：ER、1NF～3NF 分解各 ≥1 题
- [ ] 数据库：SQL 读懂增删改查；事务 ACID、隔离级别
- [ ] 网络/数据库笔记：各 ≤2 篇速查（不扩写长文）
- [ ] UML：类图关系、用例图；每周 2×30min 跟 1 道真题读图
- [ ] **月末**：范式能对；能根据描述补简单类图

### 9 月 checklist（约 30h）

- [ ] 四门选择轮转：每科每周 ≥10 题（不新开长笔记）
- [ ] 算法：快排/堆排过程、二叉树遍历；复杂度表 1 张
- [ ] 设计模式：单例、工厂、适配器、装饰、观察者、策略（场景→模式名）
- [ ] 结构化：DFD 一层 + 模块耦合/内聚
- [ ] 下午：每 2 周 1 道完整题（UML 或算法），限时 45min
- [ ] **月末**：上午四套轮转自测 ≥45/75（或连续两次 ≥42）

### 10 月 checklist（约 30h）

- [ ] 近 2 年上午选择：约 100 题（每 2 天 25 题），错题本
- [ ] 下午：UML 1 题 + 算法 1 题（拆练）
- [ ] 法规 + 标准 + 版权：2 页速查表
- [ ] 专业英语：缩写表，每天 10 个 ×7 天
- [ ] 设计模式：只做「题干场景 → 选模式」闪卡，不扩笔记

### 11 月 checklist（考前 3～4 周）

- [ ] 考前 3 周：上午整套限时 1 次/周；其余天错题 + 手算（PV/子网/范式/LRU）
- [ ] 考前 2 周：下午 1 套里选 2 题手写；法规英语表每天 15min
- [ ] 考前 1 周：只看错题 + 速查表，不做新章节
- [ ] **上考场前**：至少 2 次完整上午限时；下午四类题型各练过 1 遍
