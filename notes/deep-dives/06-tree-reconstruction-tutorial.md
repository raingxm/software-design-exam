# 二叉树遍历还原 — 由中序 + 前序/后序画树

> 对应主笔记：[数据结构回顾巩固](../03-data-structures/01-data-structures-review.md) §四  
> 来源：2026-07 第 3 章深化

---

## 1. 三种遍历 — 先记顺序

| 遍历 | 访问顺序 | 口诀 |
|------|----------|------|
| **前序** | 根 → 左 → 右 | 根在前 |
| **中序** | 左 → 根 → 右 | 根在中 |
| **后序** | 左 → 右 → 根 | 根在后 |

**手算技巧**：先标出「根」的位置，再分左右子树递归。

---

## 2. 什么组合能唯一确定一棵树？

| 组合 | 能否唯一确定？ | 原因 |
|------|----------------|------|
| 中序 + 前序 | ✅ 能 | 前序给根，中序分左右 |
| 中序 + 后序 | ✅ 能 | 后序给根，中序分左右 |
| 前序 + 后序 | ❌ 不能 | 不知道左子树有几个结点 |
| 仅前序 / 仅后序 | ❌ 不能 | 缺左右分界 |

> **中序是「地图」**：它告诉你左子树、右子树各有哪些结点。

---

## 3. 还原四步套路（通用）

以 **中序 + 后序** 为例（中序 + 前序同理，只是根的位置不同）：

```text
1. 在后序里找根（最后一个）
2. 在中序里找这根，左边 = 左子树结点，右边 = 右子树结点
3. 数清左子树有几个结点 k
4. 在后序里：前 k 个是左子树，后面是右子树 → 递归
```

**中序 + 前序** 的变体：

```text
1. 在前序里找根（第一个）
2. 在中序里分左右
3. 左子树 k 个结点 → 前序的第 2～(k+1) 个是左子树
4. 递归
```

## 4. 例题 1（软考经典）：中序 + 后序

> 中序：`D B E A C`  
> 后序：`D E B C A`  
> 画二叉树，并写前序。

### Step 1：后序最后一个 = 根

```text
后序: D E B C | A
                  ↑ 根 = A
```

### Step 2：在中序里分左右

```text
中序: D B E | A | C
      └左子树┘   └右┘
```

### Step 3：处理左子树 {D,B,E}

左子树后序 `D E B` → 根 = **B**

```text
中序: D | B | E  →  B 的左=D，右=E
```

### Step 4：处理右子树 {C}

只有一个结点 → 就是 **C**

### 画出的树

```text
        A
       / \
      B   C
     / \
    D   E
```

### Step 5：写前序（根→左→右）

```text
A → B → D → E → C
前序: A B D E C
```

## 5. 例题 2：中序 + 前序

> 前序：`A B D E C`  
> 中序：`D B E A C`

### Step 1：前序第一个 = 根 A

### Step 2：中序分左右

```text
中序: D B E | A | C
      └─左──┘     └右┘
```

左子树 3 个结点 → 前序 `B D E` 中，B 是左子树根

```text
前序左子树部分: B D E
中序: D | B | E  →  B 左=D 右=E
```

右子树：C

结果树与例题 1 相同 ✓

## 8. 为什么「前序 + 后序」不能唯一确定？

> 2026-07-08 已理解并归档

### 核心结论

还原二叉树需要两件事：

1. **谁是根** → 前序第一个 / 后序最后一个都能给 ✅
2. **左、右子树各有哪些结点** → **只有中序能给** ❌（前序/后序单独或组合都不够）

**一句话**：前序和后序能告诉你「根是谁、还有哪些结点」，但**不知道左子树有几个结点**——左右分界只有中序能提供。

### 反例 1：两结点（最小）

前序 `A B`，后序 `B A` → 根 = A，但 B 可以是左孩子或右孩子：

```text
  A        A
 /          \
B            B

前序 AB、后序 BA  —  两棵树都满足
```

若有中序：`B A` → B 在左；`A B` → B 在右 → 唯一确定。

### 反例 2：三结点

前序 `A B C`，后序 `C B A` → 根 = A，B、C 的左右分布不确定：

```text
  A          A          A
 / \          \        /
B   C          B      B
                 \      \
                  C      C

三种结构的前序、后序完全相同
```

补充中序 `B A C` → 左 {B}、右 {C} → 只有第一种成立。

### 对比：中序 + 后序为什么够用？

中序 `D B E A C` + 后序 `D E B C A`：

```text
后序最后 A = 根
中序：D B E | A | C  →  左 3 个、右 1 个
后序：D E B | C | A  →  按 k=3 切左子树，递归
```

**中序是「地图」**：根在中序里的位置 = 左右子树分界线；有了 k，才能在另一序列里切出左 k 个、右剩余。

### 还原能力对照

| 组合 | 能否唯一确定？ | 原因 |
|------|----------------|------|
| 中序 + 前序 | ✅ | 前序给根，中序分左右 |
| 中序 + 后序 | ✅ | 后序给根，中序分左右 |
| 前序 + 后序 | ❌ | 都能给根，无人能切左右 |

→ **中序遍历 = 递增序列**（软考常考判断题）

## 11. 考试速记

- **前序第一个 = 根**；**后序最后一个 = 根**
- **中序是地图**：根左边全是左子树
- **前序+后序不能唯一确定**（缺中序）
- 画完树用前序/中序/后序**互相验证**
