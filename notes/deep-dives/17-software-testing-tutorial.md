## 1. 黑盒：等价类 + 边界值

### 等价类

把输入切成「同类」区间，**每类测 1 个代表值**。

| 类型 | 怎么切 |
|------|--------|
| **有效** | 符合规格的一类（常 1 类；除非题干再按业务分档） |
| **无效** | 越下界、越上界、类型错（小数/字符）等，**分开计类** |

**易错**：`score ∈ [0,100]` 时，`<0` 与 `>100` 是**无效**，不是有效。

### 边界值

错误常在边界。闭区间 `[a,b]` 常测：

```text
a-1, a, a+1,  b-1, b, b+1
```

例：密码长度 `6≤L≤12` → `5,6,7,11,12,13`  
例：成绩整数 `[0,100]` → `-1,0,1,99,100,101`

## 2. 白盒：覆盖强度

从弱到强（必须背准顺序）：

```text
语句 < 判定(分支) < 条件 < 判定/条件 < 路径
```

| 覆盖 | 一句话 |
|------|--------|
| 语句 | 每条语句至少执行 1 次 |
| 判定 | 每个判定的 T/F 都走到 |
| 条件 | 每个原子条件的 T/F 都出现 |
| 判定/条件 | 判定覆盖 + 条件覆盖 |
| 路径 | 所有路径（最强，常爆炸） |

### 经典陷阱：条件覆盖 ≠ 判定覆盖

`if (A && B)`，用例 `(T,F)` 与 `(F,T)`：

- **条件覆盖**：✓（A、B 各有真假）
- **判定覆盖**：✗（两次 `A&&B` 都是假，真分支没走）

`if (A || B)`，要**同时**条件+判定，可选：

- `(T,T)` 与 `(F,F)` ✓  
- 注意：`(T,F)` 与 `(F,T)` 两次判定都真 → 条件✓ 判定✗

## 5. 口令卡（30 秒）

```text
边界：[a,b] → a-1,a,a+1,b-1,b,b+1
等价：越界=无效；类型错另开类
强度：语句 < 判定 < 条件 < 判定/条件 < 路径
陷阱：条件覆盖未必含判定覆盖
集成：顶向下→桩；底向上→驱动
```
