# 磁盘调度详解（SCAN / LOOK）

> 电梯式磁头调度 · 软考上午高频手算  
> 配套回顾：[操作系统第 4 章回顾 · 设备与磁盘](../02-os/04-os-review.md#七设备与磁盘管理)

---

## 一、这道题在考什么？

多个 I/O 请求落在不同磁道上，磁头一次只能停一处。调度算法决定 **服务顺序**；上午题通常只算：

> 磁头当前位置 + 请求序列 + **初始方向**（+ 常默认柱面范围 0～199）→ 服务顺序 → **总寻道距离**（相邻磁道号差的绝对值之和）

```text
存取时间 = 寻道 + 旋转延迟 + 传输
```

手算重点几乎都在 **寻道距离**。

---

## 二、先建立图像

把磁道想成门牌号排成一条线：

```text
0 -------- 14 -- 37 -- 50 -- 65-67 -- 98 -- 122-124 -- 183 -------- 199
                 ↑
              磁头在这里
```

**电梯共同规则**（SCAN 与 LOOK 都遵守）：

1. 先定当前方向（题干常写「向磁道号增大」）。
2. 沿该方向，把这一侧请求按经过顺序全部服务完。
3. **掉头**，再服务另一侧。

与 SSTF 不同：不是每一步都选「离当前最近」，而是 **整段扫完再反向**。

---

## 三、SCAN vs LOOK：唯一本质差别

| | **SCAN（电梯）** | **LOOK** |
|--|------------------|----------|
| 何时掉头 | 走到磁盘 **物理边界**（如 0 或 199） | 走到当前方向上 **最后一个请求** |
| 边界上有没有请求 | 常常没有，仍要走到头 | 不会空跑到没请求的边界 |
| 寻道距离 | 通常更大 | 通常更小 |

口诀：

> **SCAN 扫到墙；LOOK 扫到最后一个乘客就回头。**

迷你例（已掌握 · 2026-07-21）：磁头 **100**，请求只有 `30`、`150`，先向右，磁盘 0～199。

```text
LOOK：100 → 150 → 30
SCAN：100 → 150 → 199 → 30
```

---

## 四、手算模板

1. 记下：当前位置、初始方向、（若有）磁道上下限。  
2. 请求排序；分成「当前方向一侧」与「另一侧」。  
3. 先服务当前方向一侧（按扫过顺序）。  
4. **SCAN**：接到 **边界** 再掉头；**LOOK**：接到 **该侧最远请求** 就掉头。  
5. 反向服务另一侧。  
6. 相邻位置作差，求和。

---

## 五、经典例（回顾自测同序列）

磁头在 **50**，请求 `98, 183, 37, 122, 14, 124, 65, 67`，**先向磁道号增大**。  
柱面范围按教材常见设定：**0～199**。

排序后：`14, 37, 65, 67, 98, 122, 124, 183`  
右侧（>50）：`65, 67, 98, 122, 124, 183`  
左侧（<50）：`14, 37`

### LOOK（到末请求就反）

```text
50 → 65 → 67 → 98 → 122 → 124 → 183 → 37 → 14
```

| 段 | 距离 |
|----|------|
| 50→65 | 15 |
| 65→67 | 2 |
| 67→98 | 31 |
| 98→122 | 24 |
| 122→124 | 2 |
| 124→183 | 59 |
| 183→37 | 146 |
| 37→14 | 23 |
| **合计** | **302** |

### SCAN（到边界 199 再反）

```text
50 → 65 → 67 → 98 → 122 → 124 → 183 → 199 → 37 → 14
```

比 LOOK 多 `183→199`（+16）；回头从 199 起算到 37（162），不再从 183 起算（146）。

| 段 | 距离 |
|----|------|
| … 同 LOOK 直到 183 | 15+2+31+24+2+59 = 133 |
| 183→199 | 16 |
| 199→37 | 162 |
| 37→14 | 23 |
| **合计** | **334** |

**标注更正**：旧回顾笔记把路径 `…183→37→14`、距离 **302** 写成了「SCAN」——按严格定义这是 **LOOK**；0～199 下 SCAN 应为 **334**。

---

## 六、考场易错清单

1. **SCAN / LOOK 名字对调**（是否到磁盘端点）。  
2. 题给了 0～199 却按 LOOK 算 SCAN。  
3. 初始方向搞反（先服务了另一侧）。  
4. 掉头后顺序写反（应 37→14，不是 14→37）。  
5. 漏算「到边界」或「从边界回头」那一大段。  
6. 与 **C-SCAN** 混淆：到头后跳回另一端，归途通常不再服务。

---

## 七、和其他算法一眼辨

| 算法 | 一句话 |
|------|--------|
| FCFS | 按到达顺序，可能来回狂奔 |
| SSTF | 总选最近，可能饿死远端 |
| SCAN | 电梯到 **墙** |
| LOOK | 电梯到 **最后一单** |
| C-SCAN / C-LOOK | 主要只服务一个方向 |

---

## 八、速记卡

```text
LOOK：扫到最后一个请求就回头（本题 302）
SCAN：扫到磁盘边界再回头（0～199 时本题 334）
手算：方向 → 分两侧 → 扫完（+边界?）→ 反向 → 求和
```
