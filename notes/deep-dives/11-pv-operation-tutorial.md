# PV 操作详解

> 操作系统同步互斥的核心机制，软考必考

---

## 一、信号量的本质

信号量 `S` 是一个整数，表示资源状态：

| S 的值 | 含义 |
|--------|------|
| `S > 0` | 有 S 个资源空闲，没人等 |
| `S = 0` | 资源刚好用完，没人等 |
| `S < 0` | 资源透支，**\|S\| 个进程在等待** |

---

## 二、P 和 V 的语义

### P 操作（申请资源）

```text
P(S):
  S = S - 1          ← 先减
  if S < 0:          ← 再判断
    调用者【阻塞】，进入等待队列
```

### V 操作（释放资源）

```text
V(S):
  S = S + 1          ← 先加
  if S ≤ 0:          ← 再判断
    【唤醒】一个等待进程
```

### 为什么 V 判断 S ≤ 0 而不是 S ≥ 0？

关键：**V 是先加再判断**

```text
初始: S = -2  → 有 2 个进程在等待队列里

执行 V(S):
  S = -2 + 1 = -1  ← 加完后 S = -1，仍 ≤ 0
  
  S ≤ 0 说明什么？
  → 加之前是 -2，有 2 个进程等
  → 加之后是 -1，还有 1 个进程等
  → 【之前确实有人在等】→ 应该唤醒一个
```

**一句话**：V 之后 S 还 ≤ 0，说明**之前有人等**，得唤醒。

### 对照表

| 操作 | 步骤 | 判断条件 | 原因 |
|------|------|----------|------|
| P(S) | 先减：`S = S - 1` | 若 `S < 0` 则阻塞 | 减完变负 = 资源不够，得等 |
| V(S) | 先加：`S = S + 1` | 若 `S ≤ 0` 则唤醒 | 加完还 ≤ 0 = 之前有人等，得放 |

## 三、生产者-消费者：正确写法

```text
empty = n   // n 个空位
full  = 0   // 0 个产品
mutex = 1   // 互斥锁

生产者:                     消费者:
  P(empty)  ← 先抢空位        P(full)   ← 先抢产品
  P(mutex)  ← 再锁缓冲区      P(mutex)
  放入产品                    取出产品
  V(mutex)                    V(mutex)
  V(full)   ← 通知有产品了    V(empty)  ← 通知有空位了
```

**关键顺序**：先 P 资源信号量（empty/full），再 P 互斥锁（mutex）

---

## 四、为什么先 P(mutex) 会死锁？

假设**错误写法**：生产者先 `P(mutex)` 再 `P(empty)`

### 场景：缓冲区满了（empty=0, full=n）

```text
时刻 1: 生产者执行 P(mutex)
        → mutex: 1→0，生产者拿到锁，进入临界区

时刻 2: 生产者执行 P(empty)
        → empty: 0→-1，empty<0，生产者【阻塞】
        → 但它还【持有 mutex 锁】！

时刻 3: 消费者想消费，执行 P(mutex)
        → mutex: 0→-1，消费者【阻塞】
        → 消费者等生产者释放锁

结果:
  生产者等消费者取走产品（等 empty 变正）
  消费者等生产者释放锁（等 mutex）
  → 互相等待 → 【死锁】
```

### 图示

```text
生产者                      消费者
   │                          │
   ├─ P(mutex) ✓ 拿到锁       │
   │                          │
   ├─ P(empty) ✗ 阻塞 ←───────┤ 需要消费者 V(empty)
   │  （持有 mutex）           │
   │                          ├─ P(mutex) ✗ 阻塞
   │                          │  等生产者释放
   ▼                          ▼
  【死锁：循环等待】
```

### 正确顺序为什么安全？

```text
生产者先 P(empty)：
  - 若 empty=0（满了），生产者阻塞
  - 但它【还没拿 mutex】！
  - 消费者可以正常 P(full)→P(mutex)→取产品→V(mutex)→V(empty)
  - V(empty) 唤醒生产者
  - 不会死锁
```

---

## 五、一句话记忆

> **先抢资源（empty/full），再抢锁（mutex）**  
> 否则：拿着锁等资源 + 等锁才能释放资源 = 死锁

## 六、信号量初值设定

| 用途 | 初值 | 例子 |
|------|------|------|
| 互斥 | 1 | mutex = 1 |
| 同步（等待事件） | 0 | full = 0（一开始没产品） |
| 资源计数 | n | empty = n（n 个空位） |
