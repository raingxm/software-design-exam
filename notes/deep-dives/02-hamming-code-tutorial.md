## 1. 和奇偶校验、CRC 的对比

| 方式     | 典型码距 | 检错     | 纠错     | 关键词           |
|----------|----------|----------|----------|------------------|
| 奇偶校验 | 2        | 1 位     | 不能     | 知道坏了，不知位 |
| **海明码** | **3**  | 2 位     | **1 位** | 能定位并改回     |
| CRC      | 较大     | 多位     | 不能     | 模 2 除法，网络  |

## 2. 合法码字是什么？

### 比喻：带暗号的快递单

你要寄 4 个数字 `1、0、1、1`（4 位**信息位**）。

海明码规定：必须再填 3 个**校验位**，凑成 7 位，且这 7 位要满足固定规则（类似暗号）。

- **合法码字** = 按规则填好后，**能通过校验**的 7 位串
- **非法码字** = 随便写 7 位，多半通不过校验（传输出错）

### 小例子

```text
信息位（4 位）:  D1 D2 D3 D4  =  1  0  1  1

加 3 位校验位后，得到 7 位:
  P1 P2 D1 P4 D2 D3 D4  =  1  0  1  1  0  1  1
  ↑  ↑     ↑
  校验位由数据位算出来，不是随便填

「1011011」就是一个合法码字。
```

**关键点：**

- 4 位数据 → 16 种不同数据 → 16 个合法码字
- 7 位总共 128 种串，只有 16 个合法，其余 112 个都是异常

**一句话：合法码字 = 加了校验位、且校验规则说得通的数据。**

## 3. 码距是什么？

两个**合法码字** A 和 B，从 A 变成 B 至少要改几位，就是它们之间的**码距**。

```text
合法码字 A:  1 0 1 1 0 1 1
合法码字 B:  1 0 1 0 0 1 1
                  ↑
              只有 1 位不同 → A 与 B 码距 = 1

合法码字 C:  0 1 0 1 1 0 0
与 A 有 4 位不同 → A 与 C 码距 = 4
```

**整组码的码距** = 所有合法码字两两之间，那个「至少差几位」的**最小值**。

海明码任意两个合法码字至少差 **3 位** → **码距 = 3**。

## 4. 为什么码距 3 才能纠 1 位？

### 公式

| 目标       | 所需最小码距 |
|------------|--------------|
| 检错 d 位  | ≥ d + 1      |
| 纠错 t 位  | ≥ 2t + 1     |

要纠 1 位错（t = 1）：2 × 1 + 1 = **3**。

### 门牌号比喻（不用「纠错球」）

传输时**最多错 1 个数字**。

```text
本来寄的是 A:  1 0 1 1 0 1 1  （合法）
传错了 1 位:  1 0 1 1 1 1 1  （非法，第 5 位变了）
```

你要决定：它本来应该是 A，还是别的 B？

```text
码距 = 1:  A ●—● B     错 1 位，可能像 A 也可能像 B  → 纠不了

码距 = 3:  A ●———● B   错 1 位，只靠近 A，离 B 还远   → 能纠回 A
```

**口诀：要纠 t 位错，合法码字之间至少要隔 2t+1 位。**

## 5. 校验位公式：2^k ≥ n + k + 1

```text
2 的 k 次方  ≥  n + k + 1

n = 信息位个数
k = 校验位个数
n + k = 总位数
```

### 在说什么？

k 个校验位能表示 **2^k** 种不同状态（syndrome）。

收到数据后要区分：

```text
情况 0:  没出错
情况 1:  第 1 位错了
情况 2:  第 2 位错了
  ...
情况 n+k: 第 (n+k) 位错了
```

一共 **1 + (n + k) = n + k + 1** 种情况 → 必须 2^k ≥ n + k + 1。

### 手算：4 位信息要几个校验位？

```text
n = 4

k=1:  2^1 = 2,   需要 4+1+1=6  →  2 ≥ 6 ?  不行
k=2:  2^2 = 4,   需要 4+2+1=7  →  4 ≥ 7 ?  不行
k=3:  2^3 = 8,   需要 4+3+1=8  →  8 ≥ 8 ?  刚好！
```

→ **4 位信息 + 3 位校验 = 7 位**（经典 Hamming(7,4)）

8 种状态：1 种「没错」+ 7 种「第 1～7 位各错一次」。

### 再算两例

**n = 7：** k=4 时 2^4=16 ≥ 7+4+1=12 ✅ → 需 4 位校验

**n = 11：** k=4 时 16 ≥ 11+4+1=16 ✅ → 需 4 位校验

**套路：** 从 k=1 往上试，第一个让 2^k ≥ n+k+1 成立的 k 即为答案。

## 6. Hamming(7,4) 位布局

位号是 2 的幂的位置放校验位：

```text
位号:   1    2    3    4    5    6    7
       P1   P2   D1   P4   D2   D3   D4
       ↑    ↑         ↑
     校验位          校验位
```

各校验位管哪些位：

```text
位号  二进制   谁检查
 1    001     P1
 2    010     P2
 3    011     P1 + P2
 4    100     P4
 5    101     P1 + P4
 6    110     P2 + P4
 7    111     P1 + P2 + P4

汇总:
  P1 检查: 位 1, 3, 5, 7
  P2 检查: 位 2, 3, 6, 7
  P4 检查: 位 4, 5, 6, 7
```

出错后，三个校验结果拼成二进制数 = **出错位置**（P4 为高位，P1 为低位）。

## 7. 完整手算例题

### 题目

传输中**最多错 1 位**，收到：

```text
位号:   1    2    3    4    5    6    7
收到:   0    1    1    0    1    1    1
       P1   P2   D1   P4   D2   D3   D4
```

求：哪一位错了？纠正后 7 位码？原始 4 位数据？

### 步骤 1～3：重新算校验（偶校验）

偶校验：每组 1 的个数应为偶数；奇数则该组有错。

**P1 组（位 1, 3, 5, 7）：**

```text
0 + 1 + 1 + 1 = 3 个 1 → 奇数 → s1 = 1
```

**P2 组（位 2, 3, 6, 7）：**

```text
1 + 1 + 1 + 1 = 4 个 1 → 偶数 → s2 = 0
```

**P4 组（位 4, 5, 6, 7）：**

```text
0 + 1 + 1 + 1 = 3 个 1 → 奇数 → s4 = 1
```

### 步骤 4：拼错误位置

```text
s4  s2  s1  =  1   0   1  =  二进制 101  =  十进制 5
```

→ **第 5 位错了**

### 步骤 5：纠正

第 5 位 1 → 0：

```text
纠正前:  0  1  1  0  1  1  1
纠正后:  0  1  1  0  0  1  1
```

### 步骤 6：取数据位

数据在位 3, 5, 6, 7：

```text
纠正后:  1   0   1   1  →  D1 D2 D3 D4 = 1011
```

### 答案

```text
错误位置:  第 5 位（D2）
纠正后:    0 1 1 0 0 1 1
原始数据:  1011
```

## 8. 手算检查清单（考试用）

```text
1. 标位号 1～7，标 P1/P2/P4/D1～D4
2. P1 查 1,3,5,7 → 奇偶 → s1
3. P2 查 2,3,6,7 → 奇偶 → s2
4. P4 查 4,5,6,7 → 奇偶 → s4
5. 错误位 = s4 s2 s1（二进制转十进制）
6. 该位翻转（0↔1）
7. 数据 = 位 3, 5, 6, 7
```

**syndrome = 0（s4=s2=s1=0）→ 没出错。**

## 9. 自测题

### 练习 1

收到：

```text
位号:   1    2    3    4    5    6    7
收到:   1    0    1    1    0    0    1
```

<details>
<summary>答案（做完再展开）</summary>

P1(1,3,5,7): 1+1+0+1 = 3 奇 → s1=1  
P2(2,3,6,7): 0+1+0+1 = 2 偶 → s2=0  
P4(4,5,6,7): 1+0+0+1 = 2 偶 → s4=0  

syndrome = 001 = **第 1 位错**（校验位 P1，非数据位）

纠正：位 1 从 1→0 → `0 0 1 1 0 0 1`  
数据位 3,5,6,7 → **1 0 0 1**

</details>
