## 1. 软考考什么？

| 考点 | 频率 | 说明 |

| O(n log n) 时间、O(1) 空间、不稳定 | 🔥 高 | 背排序大表 |

## 2. 大顶堆 + 数组存树

**大顶堆**：每个结点 ≥ 它的孩子 → **堆顶 arr[0] 是最大值**。

```text
数组：  0   1   2   3   4   5   6
      [49, 38, 65, 13, 27, 76, 97]

树形：
              49(0)
            /       \
        38(1)       65(2)
        /  \        /  \
     13(3) 27(4)  76(5) 97(6)
```

**下标公式（必背）**：

```text
左孩子 = 2i + 1
右孩子 = 2i + 2
父结点 = (i - 1) / 2   （整除）
```

## 3. 堆排序两阶段（核心）

```text
阶段 A：初始建堆 —— 只做一次
阶段 B：排序循环 —— 每轮「换顶沉底 + 只修堆顶」，❌ 不是重新建堆
```

### 阶段 A：初始建堆（一次）

```text
1. 最后一个非叶结点 = ⌊n/2⌋ - 1
2. 从该下标减到 0，对每个结点做「下沉」(heapify)
3. 下沉规则：若 结点 < 较大的孩子 → 交换 → 继续往下沉
```

### 阶段 B：每轮排序（重复 n-1 次）

```text
Step 1：arr[0] ↔ arr[堆末] 交换  →  当前最大到末尾（已排好区）
Step 2：只对 arr[0] 下沉一次      →  修复堆顶，范围缩小 1
        ❌ 不要从 ⌊n/2⌋-1 重新建堆
```

**口诀**：**建堆一次，之后每轮：换顶沉底，只修堆顶。**

---

## 4. 为什么交换后不用「重新建堆」？

交换后只有**堆顶**可能不满足「父 ≥ 子」，其余子树结构仍在。

```text
整棵堆只有堆顶「坏了」
     ↓
只对 index 0 下沉（O(log n)）
     ↓
不用从 ⌊n/2⌋-1 全部重来（那是 O(n) 的初始建堆）
```

| | 初始建堆 | 每轮交换后 |
|---|---------|-----------|
| 范围 | 全部非叶结点 | 仅 index 0 |
| 下标 | ⌊n/2⌋-1 → 0 | 只 0 |
| 何时 | **最开始 1 次** | 每轮 1 次 |

## 5. 手算例题：完整走前三轮

序列 `{49, 38, 65, 13, 27, 76, 97}`，n=7

### 5.1 初始建堆（阶段 A，只做一次）

最后一个非叶 = ⌊7/2⌋ - 1 = **2**，从 index 2 → 1 → 0 下沉：

| 步骤 | 下沉结点 | 动作 | 数组 |
|------|---------|------|------|
| 1 | index 2 (65) | 65 < 97，交换 | `[49, 38, 97, 13, 27, 76, 65]` |
| 2 | index 1 (38) | 38 > 子，不动 | 不变 |
| 3 | index 0 (49) | 49↔97，49↔76 | `[97, 38, 76, 13, 27, 49, 65]` |

建堆完成：

```text
        97
       /  \
     38    76
    / \   /
  13 27 49  65
```

### 5.2 第 1 轮（阶段 B）

**Step 1** 堆顶 97 ↔ 末尾 65：

```text
[65, 38, 76, 13, 27, 49 | 97]
 ↑堆顶坏了              ↑已排好
```

**Step 2** 只对 index 0 下沉（范围 0～5）：

```text
65 < 76 → 交换 → [76, 38, 65, 13, 27, 49 | 97]
65 > 49 → 停止
```

### 5.3 第 2 轮

**Step 1** 堆顶 76 ↔ 当前末尾 49（index 5）：

```text
[49, 38, 65, 13, 27 | 76, 97]
 ↑堆顶坏了         ↑已排好
```

**Step 2** 只对 index 0 下沉（范围 0～4）：

```text
49 < 65 → 交换 → [65, 38, 49, 13, 27 | 76, 97]
49 是叶子 → 停止
```

### 5.4 第 3 轮

**Step 1** 65 ↔ 27 → `[27, 38, 49, 13 | 65, 76, 97]`

**Step 2** index 0 下沉：27↔49 → `[49, 38, 27, 13 | 65, 76, 97]`

继续直到堆内剩 1 个元素 → 全程有序。

### 5.5 数组视角（堆区 vs 已排好区）

```text
建堆后：  [97 38 76 13 27 49 65]           ← 全是堆区
第1轮后： [76 38 65 13 27 49 | 97]         ← 竖线右不再动
第2轮后： [65 38 49 13 27 | 76 97]
第3轮后： [49 38 27 13 | 65 76 97]
...
最终：    [13 27 38 49 65 76 97]
```

## 6. 与归并对比（空间）

| | 归并 | 堆排序 |
|---|------|--------|
| 核心 | 分两半 + **合并** | 建堆 + 堆顶换末尾 |
| 额外空间 | **O(n)** 合并要临时数组 | **O(1)** 原地交换 |
| 稳定 | ✅ | ❌ |

**堆不稳定原因**：下沉/交换时元素可**跨很远下标**移动（非相邻交换），相等元素相对顺序可能改变。

---

## 7. 与其他排序第一趟对比

同序列 `{49, 38, 65, 97, 76, 13, 27}`：

| 算法 | 第一趟/首轮特征 | 稳定？ |
|------|----------------|--------|
| 冒泡 | 97 沉底 | ✅ |
| 选择 | 13 换到最前 | ❌ |
| 插入 | 前 2 个有序 38,49 | ✅ |
| 快排 | 49 到中间分区 | ❌ |
| 堆排序 | 建堆后 97↔65，再下沉堆顶 | ❌ |

**稳定性口诀**：**「冒插归基」稳定，「选快堆希」不稳定**

---

## 8. 考试速记

- **大顶堆**：父 ≥ 子，堆顶最大
- **建堆一次**：从 ⌊n/2⌋-1 到 0 逐个下沉
- **每轮排序**：堆顶 ↔ 末尾 → **只下沉 index 0**（不重新建堆）
- **O(n log n)** 时间，**O(1)** 空间，**不稳定**

3. 堆排序和归并排序，哪个需要 O(n) 额外空间？
