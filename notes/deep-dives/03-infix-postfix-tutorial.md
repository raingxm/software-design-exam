# 中缀 ↔ 后缀（逆波兰式）深度讲解

> 对应主笔记：[程序设计语言基础知识](../02-programming-lang/01-language-fundamentals.md) §5  
> 来源：2026-06 第 2 章复习巩固

---

## 1. 三种写法是什么？

同一个式子 `3 + 4 × 5`，有三种常见写法：

| 写法 | 名称 | 例子 | 特点 |
| :--- | :--- | :--- | :--- |
| **中缀** | 运算符在中间 | `3 + 4 * 5` | 日常写法，需要括号 + 优先级 |
| **后缀** | 运算符在后面 | `3 4 5 * +` | 也叫**逆波兰式**，不用括号 |
| **前缀** | 运算符在前面 | `+ 3 * 4 5` | 也叫波兰式，软考考得少 |

**后缀的核心规则**：每个运算符紧跟在它需要的操作数**后面**。

- `3 4 5 * +` 读作：先算 `4 * 5`，再和 `3` 相加
- 等价于中缀：`3 + (4 × 5)`

---

## 2. 为什么考试要考这个？

1. **编译器**会把中缀表达式转成后缀（或类似形式），方便计算
2. **后缀求值**只需要一个栈，从左扫到右，规则固定，**不需要考虑优先级和括号**

---

## 3. 中缀 → 后缀：完整算法

准备两样东西：

- **运算符栈**：暂存还没输出的运算符
- **输出队列**：按顺序拼接结果

### 规则（按顺序判断当前符号）

| 当前符号 | 做什么 |
| :--- | :--- |
| **操作数**（a、b、3、4…） | 直接输出 |
| **左括号 `(`** | 压入运算符栈 |
| **右括号 `)`** | 弹出栈顶运算符并输出，直到弹出 `(`；`(` 丢弃，不输出 |
| **运算符** | 若栈顶运算符**优先级 ≥ 当前运算符**，先弹出栈顶并输出，再压入当前运算符 |

### 优先级（数字越大优先级越高）

```text
*  /  %   →  高（3）
+  -      →  低（2）
(         →  最低（1，只在栈内比较用）
```

**记忆口诀**：

- 操作数直接出
- 左括号进栈
- 右括号弹到左括号
- 运算符比优先级，高的先出

---

## 4. 例题 1：`a + b * c` → `a b c * +`

| 步骤 | 读入 | 动作 | 栈 | 输出 |
| :---: | :--- | :--- | :--- | :--- |
| 1 | `a` | 操作数，直接输出 | 空 | `a` |
| 2 | `+` | 栈空，压入 `+` | `+` | `a` |
| 3 | `b` | 操作数，直接输出 | `+` | `a b` |
| 4 | `*` | `+` 优先级(2) < `*` 优先级(3)，压入 `*` | `+ *` | `a b` |
| 5 | `c` | 操作数，直接输出 | `+ *` | `a b c` |
| 6 | 结束 | 弹出栈中全部 | 空 | `a b c * +` |

**关键在第 4 步**：遇到 `*` 时，栈顶是 `+`，`*` 优先级更高，所以 `*` 不能先输出 `+`，而是直接压栈。这样 `*` 会紧挨在 `b`、`c` 后面输出，保证先乘后加。

**理解**：中缀里 `b * c` 要先算，后缀里就要让 `*` 紧跟 `b c`，得到 `b c *`，最后才是 `+` → `a b c * +`。

---

## 5. 例题 2：`(a + b) * c` → `a b + c *`

| 步骤 | 读入 | 动作 | 栈 | 输出 |
| :---: | :--- | :--- | :--- | :--- |
| 1 | `(` | 左括号入栈 | `(` | 空 |
| 2 | `a` | 操作数，直接输出 | `(` | `a` |
| 3 | `+` | 压入 `+` | `( +` | `a` |
| 4 | `b` | 操作数，直接输出 | `( +` | `a b` |
| 5 | `)` | 弹到 `(`：`+` 输出，`(` 丢弃 | 空 | `a b +` |
| 6 | `*` | 栈空，压入 `*` | `*` | `a b +` |
| 7 | `c` | 操作数，直接输出 | `*` | `a b + c` |
| 8 | 结束 | 弹出 `*` | 空 | `a b + c *` |

**关键在第 5 步**：`)` 强制把括号里的 `+` 先输出，得到 `a b +`，相当于 `(a+b)` 已经算完。然后 `*` 和 `c` 接在后面，表示再乘 `c`。

**对比两题**：

| 中缀 | 后缀 | 含义 |
| :--- | :--- | :--- |
| `a + b * c` | `a b c * +` | a + (b×c) |
| `(a + b) * c` | `a b + c *` | (a+b) × c |

括号的作用，在后缀里变成了**输出顺序**的变化。

---

## 6. 后缀求值：用栈从左到右扫

规则极其简单：

1. 遇到**数字** → 入栈
2. 遇到**运算符** → 弹出**两个**操作数（先弹的是右操作数），算完结果再入栈
3. 扫完后，栈里剩一个数就是答案

### 例题：`3 4 5 * +` = ?

从左到右扫描：

| 步骤 | 读到 | 栈的变化 | 说明 |
| :---: | :--- | :--- | :--- |
| 1 | `3` | `[3]` | 入栈 |
| 2 | `4` | `[3, 4]` | 入栈 |
| 3 | `5` | `[3, 4, 5]` | 入栈 |
| 4 | `*` | `[3, 20]` | 弹出 5、4 → 4×5=20，入栈 |
| 5 | `+` | `[23]` | 弹出 20、3 → 3+20=23，入栈 |

**答案：23**

对应中缀：`3 + 4 × 5`（先乘后加）

### 注意：减法和除法的顺序

- 后缀 `8 3 -`：先弹 3（右），再弹 8（左）→ `8 - 3 = 5`
- 后缀 `8 3 /`：→ `8 / 3`

> **🔥 考点提醒**：先弹出的是**右操作数**，这是软考手算常考的坑。

---

## 7. 再练一题（带括号的中缀转后缀）

`(3 + 4) * 5` → ?

| 步骤 | 读入 | 输出 |
| :---: | :--- | :--- |
| `(` | 入栈 | |
| `3` | 输出 | `3` |
| `+` | 压栈 | `3` |
| `4` | 输出 | `3 4` |
| `)` | 弹出 `+` | `3 4 +` |
| `*` | 压栈 | `3 4 +` |
| `5` | 输出 | `3 4 + 5` |
| 结束 | 弹出 `*` | `3 4 + 5 *` |

验证求值：`3 4 + 5 *` → (3+4)×5 = **35** ✓

## 8. 一张图串起来

```text
中缀:  a  +  b  *  c
       ↓     ↓  ↓  ↓
后缀:  a     b  c  *  +
              └──┘     └── 先算 b*c，再和 a 加

后缀:  3  4  5  *  +
       ↓  ↓  ↓  ↓  ↓
栈:   [3][4][5]→20→23
```

---

## 9. 考试速记

**中缀 → 后缀**：操作数直接出；`(` 入栈；`)` 弹到 `(`；运算符比优先级。

**后缀求值**：遇数入栈，遇运算符弹两个算完再入栈。
