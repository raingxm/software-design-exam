## 一、口令（够用了）

```text
说明圈名字 → 图上对人、对箭、对加工 → 没有就补
父子平衡   → 只对「进出这道加工」的进出口对账；肚子里的箭不算
```

**四要素（人话）**

| 叫什么 | 是什么 | 图上大致样子 |
|--------|--------|--------------|
| 外部实体 | 系统外的人/组织/别的系统 | 方框 |
| 加工 | 系统里干活的一步（数据含义会变） | 圆角框 / 圆 |
| 数据存储 | 数据停放处（表、文件） | 两条横线 |
| 数据流 | 正在移动的、**有名字**的数据 | 带名箭头 |

## 三、核心考点清单（有没有漏）

### 已覆盖（P0）

- [x] 四要素是什么  
- [x] **缺流**：说明有单据往来，图上少箭  
- [x] **缺外部实体**：说明有外头的人/系统，图上没方框  
- [x] **父子平衡**：父加工进出口 vs 子图边界；内部箭不影响  

### 知道即可、今晚未单练（P1，真题里仍会出现）

| 点 | 一句话 | 考试怎么用 |
|----|--------|------------|
| **缺加工** | 说明里有一道「办事步骤」，图上没这个处理框 | 下午补加工名；和缺流一起找 |
| **连线禁区** | 两个外部实体不能直接连；两个存储不能直接连；实体↔存储一般也要经加工 | 上午偶考「哪张图不合法」 |
| **加工要有进有出** | 处理框不能只有进或只有出 | 残图常用来暗示还缺一条流 |
| **顶层 / 0 层** | 整个系统画成**一个**加工，四周全是外部实体 | 认识即可；细活在 1 层及以下 |
| **数据字典** | 给数据流/存储写条目定义（由哪些数据项组成） | 上午轻量；下午偶见填空 |
| **分层编号** | 父 1 → 子 1.1、1.2… | 看懂编号对应哪层 |

## 四、下午题侦探顺序（实战）

1. 读说明，圈：**谁**（实体）、**什么单**（流名）、**哪一步**（加工）、**存哪**（存储）  
2. 对残图：人齐不齐 → 箭齐不齐 → 加工齐不齐  
3. 若给了父子两层：用进出口清单对账（多了 / 少了）  
4. 不确定时：优先信**说明文字**，图是残的
