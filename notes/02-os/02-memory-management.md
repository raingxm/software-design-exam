# 存储管理 (Memory Management)

存储管理主要负责内存的分配、保护、扩充和地址映射。

## 1. 地址映射 (Address Mapping)

- **逻辑地址 (Logical Address)**：程序中使用的相对地址。
- **物理地址 (Physical Address)**：内存中的实际地址。
- **重定位 (Relocation)**：将逻辑地址转换为物理地址的过程。

---

## 2. 存储分配方式

### 2.1 页式存储 (Paging)
将内存分为固定大小的 **页框 (Page Frame)**，程序分为同样大小的 **页 (Page)**。
- **优点**：利用率高，无外部碎片。
- **缺点**：增加了开销；可能产生内部碎片。
- **地址结构**：页号 + 页内偏移。

### 2.2 段式存储 (Segmentation)
按程序的逻辑结构划分为若干个 **段 (Segment)**，各段大小不等。
- **优点**：逻辑独立，利于共享和保护。
- **地址结构**：段号 + 段内偏移。

### 2.3 段页式存储 (Segmented Paging)
结合两者优点：先分段，段内再分页。
- **地址转换**：段表 -> 页表 -> 物理内存。

---

## 3. 虚拟存储技术 (Virtual Memory)

基于程序的 **局部性原理 (Principle of Locality)**。

### 页面置换算法
当内存缺页且页框已满时，需要选择一页换出：
1. **最佳置换算法 (Optimal)**: 理论最优，淘汰以后最长时间不再访问的页。
2. **先进先出 (FIFO)**: 可能产生 **Belady 异常**（分配页框越多，缺页率反而越高）。
3. **最近最久未使用 (LRU)**: 性能接近 Optimal，是实际应用中最常用的。

## 4. 磁盘管理

- **存取时间** = 寻道时间 + 等待时间 (旋转延迟) + 传输时间。
- **磁盘调度算法**:
    - FCFS (先来先服务)
    - SSTF (最短寻道时间优先)
    - SCAN (电梯算法)
    - LOOK
